gulpを回そう

それぞれの簡単な説明

Node.js
ブラウザでしか使えなかったjsを、サーバーサイドでもjs実行させようというもの。
特徴
- ノンブロッキングi/o データベース参照の時にいちいち止まらないから早い
- イベントループ
npm
Node Packeged Moduules つまりnode.jsのパッケージ管理ツール。
package.jsonファイルをつくってそこで管理する。
gulp.js
node.jsをベースとしたビルドシステムヘルパー。gulp.fileにタスクを書く。

インストール方法

  1. node.jsのヴァージョンチェック

    node -v

  2. package.jsonの作成

    自力で作成しても、コマンドで作成してもいい。これでnpmによってパッケージのインストールを行います

    npm init

  3. gulpのインストール

    gulpはグローバルにもローカルにもインストールが必要。
    ローカルにgulpがあることでバージョンがプロジェクトによって違ってもタスクを実行できる。

    npm install --save-dev gulp

  4. gulpのチェック

    下のコマンドでバージョンが出てくれば完了

    gulp -v

ローカルインストールとグローバルインストール

npmはローカルとグローバルでインストールする場所を選べる。
ローカルにインストールしたパッケージは、そのプロジェクトのフォルダ内でのみ利用可能。 グローバルにインストールした場合は、そのマシン上のどこからでも利用できるようになる。 デフォルトでは、ローカルにパッケージがインストールされるようになってる。

グローバルインストール

npm install -g パッケージ名

ローカルインストール

npm install パッケージ名

save-devについて

package.jsonでインストールするパッケージの管理をするにはnpm install[-S]を実行する際に--saveまたは--save-dev[-D]というオプションを付ける。 --saveは作成しているプログラム自体にそのパッケージが必要なときに。 gulpやgulpで使うプラグインなど、開発時だけ必要となるものには--save-devを付けるとよいでしょう。

以下二つは同じ。下は省略形

npm intsall --save-dev パッケージ名
npm intsall -D パッケージ名
